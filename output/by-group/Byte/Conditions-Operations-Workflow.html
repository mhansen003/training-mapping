Required Submission Notes

Processors must document any challenging items for the Underwriter:

Type relevant notes for the Underwriter

Attachment Methods

Drag and drop documents from:

  • Stored Documents - Documents already in the loan file
  • File Explorer - Documents from your local machine

Viewing Attached Documents

Click on the file folder icon to view documents attached to a condition. Multiple documents can be attached to one condition.

Document Attachment Rule

Once a document is attached to a condition, it cannot be attached to another condition. If a document can satisfy multiple conditions, color coding must be used instead.

5

Follow Up Flag Color Coding

Communication system between Processors and Underwriters
Double-Click Condition
->
Select Follow Up Flag Color
->
Click Append Notes

Red Flag

Items submitted to underwriting and unable to clear condition(s). Underwriter must remove the received date on the condition.

Blue Flag

Processor take notice. The Processing Manager uses this flag when leaving notes on conditions when assisting with a loan.

Yellow Flag

Processor take notice. Underwriters use this flag when leaving notes on conditions for something other than not clearing it.

Green Flag

Underwriter take notice. Processors use this flag when leaving notes on conditions for the Underwriter.

Orange Flag

Processor would like to move condition to a PTF (Prior to Funding) condition.

Purple Flag

Underwriter take notice. The Underwriting Manager uses this flag when leaving notes on conditions when assisting with a loan.

Important: Remove Highlights

The user must remove the highlight once the item has been addressed to maintain accurate condition tracking.

CTC Requested Actions

Clicking the CTC Requested radio button:

  • Updates pipelines for management visibility
  • Sends notification to the Loan Officer
  • Records the CTC request date

Underwriter Note - CTC Cannot Be Granted

If CTC Requested is checked and the loan cannot be moved to CTC, the Underwriter must remove the CTC Requested date to maintain accurate pipeline tracking.

Conditions for Operations - Workflow Markdown

# Byte Conditions for Operations

## Overview
This workflow covers the complete process for underwriting submission, condition management, and clear to close procedures in Byte LOS.

## 1. Submission Notes
- Open Cover Letter - Submission Notes Screen
- Document challenging income, work history, assets, credit items
- Hit Enter to add timestamped notes (auto-saved)
- Paper option: Add "See Cover Sheet in Stored Docs"

## 2. Submit to Underwriting
- Open Status Screen
- Click "UW Received" to submit
- All Retail pipelines reflect the update

## 3. CLA Distribution
- Underwriter creates PDF of Conditional Loan Approval
- Email to: Loan Officer, Production Assistant, Processor
- Stored in: Conditional Commitment (Stored Documents)

## 4. Adding Documents to Conditions
- Open Conditions Screen
- Drag & drop from Stored Documents or File Explorer
- Click file folder to view attachments
- Note: One document = One condition only

## 5. Color Coding System
| Color  | Meaning                           |
|--------|-----------------------------------|
| Red    | Unable to clear (UW action)       |
| Blue   | Processing Manager notes          |
| Yellow | UW notes (not clearing related)   |
| Green  | Processor notes for UW            |
| Orange | Move to PTF request               |
| Purple | UW Manager notes                  |

## 6. Resubmission Process
- Processor: Add docs -> Mark Received
- Processor: Request -> Mark Requested
- Underwriter: Review -> Mark Cleared

## 7. Clear to Close
- Move to "Conditions for Review" status
- Click "CTC Requested" when ready
- LO receives automatic notification
- UW removes CTC date if cannot proceed
