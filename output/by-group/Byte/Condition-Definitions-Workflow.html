Key Concept: Internal vs External

Internal conditions (checkbox checked) are NOT printed on documentation sent externally. External conditions (checkbox unchecked) ARE included on printed documentation like closing instructions.

Suspense Requirements

  • * Can be internal or external conditions
  • * ALL must be satisfied before Approval
  • * Underwriter responsibility to clear
  • * Blocks loan from being Approved status

Pro Tip

Suspense conditions are the first gate - focus on clearing these quickly to move the loan to underwriting approval and keep the pipeline flowing.

Critical Rule

ALL PTD conditions must be satisfied before a loan can be marked Clear To Close by the Underwriter. No exceptions without management override.

Responsibility Assignment

PTD conditions are cleared by Underwriter or Processor 3 in the system. The responsibility field indicates who has authority to clear each specific condition.

PTF Key Points

  • * Can be internal or external
  • * UW, Closer, or Funder responsibility
  • * Print on closing instructions (if not Internal)
  • * Cannot move to Funds Ordered with open PTF

Responsibility Rules

  • * Only UW can move UW conditions to PTF
  • * UW can update responsibility to Closer/Funder
  • * Closers review and clear during closing prep
  • * Funder clears outstanding PTF after package receipt

PTF Decision Flow

PTF Condition Not Satisfied at Funding?
Dry/Escrow State -> UW may add PTF for UW review, must satisfy before Funds Ordered
Wet Funding State -> Manager involvement required to move to Post Funding
Standard -> Responsible party moves condition to PTP

Retail UW Funding Conditions Pipeline

This pipeline displays any loan with open PTF conditions and should be monitored by all Underwriters, Processors, Closers, and Funders. Use the role and name filters to locate specific loans.

Closing Agent Contact Required

Once the funding package is received, the Funder must review and clear any outstanding PTF items and contact the Closing Agent if documentation is missing.

PTP (Prior To Purchase)

Conditions for post-closing collection

PTP Characteristics

  • * Internal or external conditions
  • * Collection at closing table
  • * Documented after funding
  • * Typically have timeframe associated

Responsibility & Printing

  • * Responsible party must clear accordingly
  • * Print on closing instructions (if not Internal)
  • * Only UW can move UW conditions to PTP
  • * Closers mark unwanted print items as Internal

Pro Tip for Closers

Items you do NOT want to print on closing instructions must be marked as Internal. Review all PTP conditions before sending closing package.

Package Receipt Process

Once the funding package is received, the Closer/Funder reviews the package for any outstanding PTP items and contacts the Closing Agent if documentation is missing.

Critical: Manager Approval Required

Only a Manager may approve moving conditions to Post Funding status. This is reserved for emergency situations where the loan must close to meet customer service expectations.

Expiration Date Example

If the borrower is starting their new job on 4/1/2020, the paystub condition expiration would be 4/1/2020. This date is used for condition tracking for the anticipated due date.

Post Funding Workflow

Processor Responsibilities
Collect and review Post Funding conditions received
Track items on Retail Open Post Funding Conditions Excel Report
Upload documentation and mark received
Add Outlook calendar reminder for outstanding conditions on due date
Underwriter Responsibilities
Monitor Retail UW Post Funding Pipeline with received dates
Clear conditions when documentation is satisfactory

30-Day Aging Limit

Post Funding conditions cannot be aged more than 30 days. These affect the salability of the loan. Corrective action will be taken if conditions are not received and updated in that timeframe.

Mandatory Documentation

If a condition is moved from one stage to another, condition notes must be added in the condition stating WHY and WHO provided approval.

Edit Condition Dialog Fields

  • Condition: Description text
  • Class: UW, PTF, etc.
  • Type: Document type (PROP, INC, CRED, etc.)
  • Stage: PTD, PTF, PTP, Post Funding, Suspense
  • Responsibility: Who can clear
  • Internal: Checkbox for print exclusion
  • Dates: Requested, Received, Submitted, Cleared
  • Expiration Date: Due date for tracking
  • Notes: Append notes with approval documentation

Quick Reference Card

CONDITION STAGES (in order):
==================================
SUSPENSE    -> Before Approval (UW responsibility)
PTD         -> Before Clear To Close (UW/Processor 3)
PTF         -> Before Funds Ordered (UW/Closer/Funder)
PTP         -> Before Purchase/Ship (with timeframe)
POST FUNDING -> After Closing (Manager approval only)

KEY RULES:
==================================
* Internal = NOT printed on external docs
* External = Printed on closing instructions
* Double-click condition to edit
* Notes REQUIRED when moving stages (why + who approved)
* Only UW can move UW conditions to PTF/PTP
* Post Funding = Manager approval + 30-day max aging

PIPELINES TO MONITOR:
==================================
* Retail UW Funding Conditions Pipeline (PTF)
* Retail UW Post Funding Pipeline
* Retail Open Post Funding Conditions Excel Report

RESPONSIBILITIES:
==================================
Underwriter  -> Suspense, PTD, PTF (UW items)
Processor 3  -> PTD
Closer       -> PTF, PTP
Funder       -> PTF, PTP
Manager      -> Post Funding approval
