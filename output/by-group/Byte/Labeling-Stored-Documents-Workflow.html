Document Status

Reviewed, Approved, or Inactive

Reviewed

  • OK Items for Underwriter review
  • OK Acceptable to remain in file
  • OK Qualifying documents pending UW approval

Approved

  • OK Final documents
  • OK Part of investor package
  • OK UW has reviewed and accepted

Inactive

  • X Expired documents
  • X Invalid documents
  • X Incomplete documents

Important: Approved Status

Qualifying documents for underwriting review should NOT be marked Approved until the Underwriter reviews and accepts them. Marking documents Inactive eliminates them from the investor package.

Status Selection Logic

Is document final?
YES -> Mark as Approved (investor package)
NO - Pending UW -> Mark as Reviewed
NO - Expired/Invalid -> Mark as Inactive

Expiration Dates

Track document validity with expiration alerts

Expiration Date Feature

Expiration dates can be used for credit or verification items for quick reference. When expiration dates are added to documents, the date shows in Stored Documents and is highlighted when the date is reached.

Expiration Tracking Flow

Open Document Properties
->

->
Date Displays in Stored Docs List
->
Date Reached?
YES ->
Document Highlighted
NO ->
Date Displays Normally

Best Practice: Use Expiration Dates For

  • Credit reports (typically 120 days)
  • Verification of Employment (VOE)
  • Bank statements (aging requirements)
  • Appraisals (typically 120-180 days)
  • Rate locks and commitments

Quick Reference Card

LABELING FORMAT:
  W2           -> B1 2019 or [Name] 2019

STATUS MEANINGS:
  REVIEWED  -> Pending UW review, acceptable in file
  APPROVED  -> Final, part of investor package (UW accepted)
  INACTIVE  -> Expired, invalid, or incomplete (excluded from package)

MODIFY DOCUMENT: Double-click in Stored Documents -> Edit fields -> Save

EXPIRATION DATES:
  - Use for credit, VOE, time-sensitive items
  - Highlighted in list view when date is reached
  - Quick reference for document validity

CATEGORIES: INC (Income), ASSET, and others per document type

# Byte Labeling Stored Documents Workflow

## Overview
Stored Documents is used by various departments during the loan process. For efficiency of file movement and review, documents can easily be labeled when storing and reviewing.

## 1. Adding a New Document
1. Drag and Drop or print the documents to Byte
2. Note: When dragging multiple documents, drag into the overview pane to split documents
3. The Import and Edit Documents window opens

## 2. Labeling Fields

### Document Type
Select from dropdown - auto-updates Description field

### Description Examples
| Type | Format | Example |
|------|--------|---------|
| W2 | Borrower + Year | B1 2019 or Francesca 2019 |
| Paystub | Borrower + Date | B1 Paystub 9.18 |
| Bank Statement | Bank + Acct + Period | TD Bank #5541 August |

## 3. Document Status

### Reviewed
- Items for Underwriter review
- Acceptable to remain in file
- Qualifying documents pending UW approval

### Approved
- Final documents
- Part of investor package
- UW has reviewed and accepted

### Inactive
- Expired documents
- Invalid documents
- Incomplete documents

**Note:** Qualifying documents should NOT be marked Approved until Underwriter reviews and accepts them.

## 4. Modifying Existing Documents
1. Double-click document in Stored Documents
2. Edit the Description and other fields
3. Save changes

## 5. Expiration Dates
- Use for credit or verification items
- Date shows in Stored Documents list
- Highlighted when expiration date is reached
