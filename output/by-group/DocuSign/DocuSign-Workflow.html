What is DocuSign?

DocuSign is CMG Financial's electronic signature platform that enables secure, legally binding digital signatures on loan documents. Key features include:

  • ESIGN Act and UETA compliant electronic signatures
  • Multi-factor authentication options for signer verification
  • Real-time tracking of document status and completion
  • Integration with Byte LOS for automated document delivery
  • Audit trail and tamper-evident seal on all documents
Document Type Typical Use Case Authentication Required
Initial Disclosures LE, Intent to Proceed, eConsent Standard Email
Application Documents 1003, 4506-C, Authorization Forms Standard Email
Closing Documents Note, Security Instrument, CD Phone or Access Code
Post-Close Corrections Affidavits, Correction Agreements Per Investor Requirements

Pro Tip: Multiple Borrowers

When adding multiple borrowers, use signing order to ensure documents are signed in sequence. This is especially important for:

  • Married couples signing together
  • Co-borrowers in different locations
  • Documents requiring witness signatures

Phone Authentication Setup

When adding a recipient:

  • Click "Identity Verification" next to recipient name
  • Select "Phone Authentication"
  • Enter the borrower's phone number (digits only)
  • Choose SMS or Voice Call delivery method
  • Borrower enters code before accessing documents

->

Setting Description
Code Format Alphanumeric, 6-50 characters (case-sensitive)
Code Delivery Must be shared via separate channel (phone call, text, etc.)
Failed Attempts After 3 failed attempts, recipient is locked out
Reset Access Sender can correct/resend to unlock recipient

Important: Code Security

Access codes must NEVER be sent in the same email as the DocuSign invitation. Always communicate the code through a separate channel such as a phone call, text message, or in-person. This two-factor approach ensures document security.

Access Code Best Practices

  • Use memorable codes related to the borrower (last 4 of SSN + loan amount)
  • Document the code shared in loan notes for reference
  • Call borrower to verify they received both email and code
  • For closings, coordinate code delivery with closing agent

Field Properties

Each field can be configured with:

  • Required: Signer must complete to finish signing
  • Read Only: Pre-filled, cannot be changed by signer
  • Conditional: Shows/hides based on other field values
  • Validation: Format rules (date, number, SSN, etc.)

Quick Tip: Auto-Place Fields

Use DocuSign's Auto-Place feature to automatically detect and place fields based on anchor text in your documents. For example, "Sign Here" or "Date" text can trigger automatic field placement, saving significant time on large document packages.

Envelope Status Definitions

Status Meaning Action Required
Sent Email delivered, awaiting recipient action Monitor for activity
Delivered Recipient opened email or accessed documents None - signing in progress
Completed All recipients have signed Download and store documents
Declined Recipient refused to sign Contact borrower, investigate issue
Voided Sender cancelled the envelope Create new envelope if needed
Expired Envelope passed expiration date without completion Resend or create new envelope

Tracking Options

  • Manage Tab: View all sent envelopes and their current status
  • History: Detailed activity log showing each action taken
  • Notifications: Email alerts when status changes
  • Reminders: Automated follow-up emails to unsigned recipients

Reminder Settings

Configure automatic reminders:

  • First reminder: 3 days after sending
  • Repeat every: 3-5 days
  • Expiration: 30-120 days (per document type)
  • Custom message: Include LO contact info and urgency

Mobile Signing

Borrowers can sign from any device:

  • Smartphone: Responsive design works on all mobile browsers
  • Tablet: Optimized for larger touch screens
  • DocuSign App: Available for iOS and Android
  • Desktop: Full experience on any web browser

Prepare Your Borrowers

Before sending DocuSign, inform borrowers:

  • Email subject will be from "DocuSign" or "CMG Financial"
  • Have their access code ready (if applicable)
  • Allow 15-30 minutes to review and sign
  • Use a stable internet connection
  • Contact LO if documents don't load properly

Troubleshooting

Common issues and resolutions
Issue Cause Resolution
Email not received Spam filter, typo in address Check spam/junk folder; verify email address; resend envelope
Access code not working Typo, case sensitivity, locked out Re-enter carefully (case-sensitive); after 3 fails, correct and resend
Phone auth code not received Wrong number, carrier blocking Verify phone number; try voice call instead of SMS
Documents won't load Browser issue, slow connection Clear cache; try different browser; use mobile app
Signature doesn't appear Browser compatibility Use Chrome or Edge; disable popup blockers
Envelope expired Signing deadline passed Create and send new envelope with fresh documents
Recipient locked out Too many failed auth attempts Go to Manage > Correct > Update recipient info and resend
Wrong signer signed Shared email/device Void envelope; send new one with phone authentication

Critical: Voiding Envelopes

If you need to void (cancel) an envelope:

  • Go to Manage > Find the envelope > More > Void
  • Enter reason for voiding (required)
  • Recipients will be notified the envelope is cancelled
  • Voided envelopes cannot be reactivated - create new

Correcting Envelopes

If you need to make changes to a sent envelope:

  • Before any signatures: Full correction available (recipients, documents, fields)
  • After partial signing: Can only correct unsigned recipients
  • After completion: Must void and create new envelope
