üîÑ Push/Sync Issues

Common issues when pushing data between Byte and DocuTech.

Push to Byte Failed
‚ñº

Documents generated in DocuTech but won't push to Byte for e-signing. May show error message or timeout.

‚úÖ Solution
  1. Wait 2-3 minutes and try pushing again (temporary connectivity issue)
  2. Verify the loan is still open in Byte and not locked
  3. Check that borrower email addresses are valid (no typos)
  4. Clear browser cache and try again
  5. If persists, contact DocuTechSupport@cmgfi.com with loan number
‚ùå
Data Not Syncing from Byte
‚ñº

DocuTech shows old loan data that doesn't match current Byte information.

‚úÖ Solution
  1. In DocuTech, click Refresh or Re-Pull Data from Byte
  2. Verify all changes are saved in Byte before creating package
  3. Close and reopen the DocuTech session
  4. Generate a new package (don't edit the old one)

Signed Docs Not Pushing Back
‚ñº

Borrower completed e-signing but documents aren't appearing back in DocuTech.

‚úÖ Solution
  1. Wait 15-30 minutes for automatic sync to complete
  2. Check Byte e-sign status to confirm all signatures are complete
  3. In DocuTech, manually check the package status
  4. If still missing after 1 hour, contact support for manual retrieval

üìÑ Document Errors

Issues with document generation and content.

‚ùå
Missing Documents in Package
‚ñº

Expected documents are not appearing in the generated package.

‚úÖ Solution
  1. Review the document selection screen - ensure all needed docs are checked
  2. Some documents are loan-type specific (FHA, VA, Conv) - verify product code is correct
  3. State-specific documents require property state to be set correctly in Byte
  4. Search the document library manually and add missing docs
‚ùå
Incorrect Data on Documents
‚ñº

Documents show wrong borrower info, loan terms, or fees.

‚úÖ Solution
  1. Correct the data in Byte first (DocuTech pulls from Byte)
  2. Generate a new package - don't try to edit the existing one
  3. Preview documents before pushing to verify corrections
  4. For fee discrepancies, check both fee worksheet AND loan terms in Byte
‚ö†Ô∏è
Document Generation Error
‚ñº

DocuTech throws an error when trying to generate documents.

‚úÖ Solution
  1. Note the exact error message for troubleshooting
  2. Check for missing required fields in Byte (common: SSN, DOB, property address)
  3. Try a different browser or clear cache
  4. If error mentions specific field, verify that field in Byte
  5. Report persistent errors to DocuTechSupport with screenshot

‚úçÔ∏è E-Sign Problems

Issues with electronic signature delivery and completion.

‚ùå
Borrower Didn't Receive E-Sign Email
‚ñº

Borrower says they never got the e-signing invitation email.

‚úÖ Solution
  1. Have borrower check spam/junk folder
  2. Verify email address is correct in Byte (common typos: .con vs .com)
  3. In Byte e-sign status, resend the invitation
  4. Try an alternate email address if available
  5. Some corporate email filters block signing emails - try personal email
‚ö†Ô∏è
E-Sign Link Not Working
‚ñº

Borrower clicks the link but gets an error or can't sign.

‚úÖ Solution
  1. Links expire after a set time - resend if more than 48 hours old
  2. Have borrower try a different browser (Chrome works best)
  3. Ensure borrower isn't on a VPN or corporate network that blocks
  4. Clear browser cache and cookies, then try link again
  5. If package was voided/cancelled, a new package must be sent
‚ÑπÔ∏è
Co-Borrower Can't Sign
‚ñº

Primary borrower signed but co-borrower's link isn't working or wasn't sent.

‚úÖ Solution
  1. Verify co-borrower has a separate email address in Byte
  2. Co-borrowers often sign sequentially - primary must complete first
  3. Check e-sign status to see if co-borrower invitation was sent
  4. Resend invitation specifically to co-borrower if needed
