What to Look For

Check the "Confirmation of Searches Performed" section of the seller's report. Look for checkmarks in both:

  • FMEL - Freddie Mac Exclusionary List
  • Other Gov't - Must include SCP (FHFA Suspended Counterparty Program)

Shortcut

If the seller's report shows both FMEL and "Other Gov't" (which includes SCP) are checked, you can skip the manual parties process. Simply upload their report to Stored Docs.

Name Entry Rules

  • Enter names exactly as written on documents
  • Include middle initials (John C. Doe, not John Doe)
  • Multiple name variations go on multiple lines
  • Leave fields blank if not applicable (e.g., no seller for refinance)

Field Flexibility

Parties don't have to match their corresponding field exactly. If a field is full, use any available field. LoanSafe screens ALL names on the Parties screen regardless of which field they're in.

Locked Fields

The "Title Company" and "Homeowner's" fields are locked for other purposes. Use alternative fields for this information.

When Parties Screen is Full

If all fields are used:

  1. Replace existing names with remaining parties
  2. It's OK if some names are listed twice
  3. Note in conversation log which names couldn't be added
  4. Credit auditors will ensure all parties are run

If Hits Are Found

If any party has a hit on the LoanSafe report:

  1. Perform due diligence to determine if the hit matches the party on the loan
  2. If confirmed match, present information to management for further review
  3. Document all findings in the file

Final Documentation

  • FHLMC Exclusionary Form - Upload under "Checklist" in Stored Docs
  • LoanSafe Report - Credit auditors label as "Fraud Report"
  • Confirmation of Searches - Print and upload as "LoanSafe Report"

Troubleshooting

Common issues and solutions

Error: "Process cannot access the file"

This error occurs occasionally when running LoanSafe:

  1. Click OK on the error message
  2. When prompted "Do you want to continue running the application?" select No
  3. This will close Byte - open a new session and try again
  4. There is currently no workaround for this issue

Second Run Required

Check the conversation log to see if Source Point couldn't fit all parties. If names are listed as not pulled:

  1. Add missing names to ANY available field on Parties screen
  2. Re-run the LoanSafe report
  3. Previously pulled names will run again (this is OK)

Quick Reference Card

CORRESPONDENT LOANSAFE WORKFLOW
================================

STEP 1: CHECK SELLER'S REPORT
- If BOTH FMEL and SCP checked -> Upload report, done
- If missing SCP -> Continue to full process

STEP 2: COMPLETE FORM
- FHLMC Exclusionary and FHFA SCP Parties Form
- Enter names EXACTLY as on documents
- Include middle initials, LLC, Inc., etc.

STEP 3: PARTIES SCREEN
- Open Byte > All Screens > Parties
- Enter all names from form
- Use any available field if correct one is full

STEP 4: RUN LOANSAFE
- Interfaces > Fraud Detection > CoreLogic Fraud
- Click Submit (or Defaults first if new user)
- Report auto-uploads to Stored Docs

STEP 5: REVIEW & DOCUMENT
- Launch LoanSafe Connect
- Click Confirmation of Searches
- Print to Stored Docs > Mark Approved
- If hits found -> Due diligence + escalate

KEY PARTY SOURCES:
- Borrowers: Name Affidavit
- Appraiser: License page (NOT signature)
- AMC: Appraisal Lender/Client section
- Title: Schedule A of Title Commitment
- Settlement: Final HUD contact info
- Seller: Final HUD transaction info
