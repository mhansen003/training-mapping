üîê Access VOE Truv Screen

1

Open the Loan in Byte

Navigate to the loan file in Byte LOS.

2

Access VOE Truv Screen

Go to Verifications ‚Üí VOE Truv in the screen menu. Note: Do NOT use the standard Verifications screen/tool for Truv orders.

‚ö†Ô∏è Important - Always Use LOS

As of 7/17/2025, reports should NOT be ordered directly via the CMG SSO menu. Orders placed via SSO will require manual Reference ID entry and won't autoflow to AUS screens or doc manager.

üìù Order Information

1

Select Application & Borrower

Use the dropdown menus to select the application and borrower for the VOE order.

2

Verify Auto-Populated Info

Borrower information auto-populates from the URLA. Verify First Name, Last Name, Mobile, Email, and SSN. Updates save back to URLA.

3

Configure SMS (Optional)

Toggle Add SMS to enable text message invitations. Orders with Email + Mobile complete approximately 2x faster.

üì± SMS Note

If your borrower does not want to receive text messages, do not click Add SMS in the New VOE Order. They will receive email-only invitations.

üîç Coverage Check

1

Select Employer/Provider

Select "Yes" under Specify Employers/Providers in the Order, then search for the borrower's employer.

2

Review Coverage Indicator

Check the color-coded coverage indicator before selecting. Proceed only if coverage is High, Low, or Not Verified (contact borrower for payroll provider info).

High Expected to succeed - send order
Low May succeed if borrower knows payroll provider
Not Verified Ask borrower about payroll provider
Not Supported Use alternative verification method
üí° Multiple Employers

You can add additional employers as needed. For previous employers, best practice is to create a separate order to avoid delaying the current employer report.

üì§ Submit & Manage

1

Click Submit

Click Submit to process the order. Borrower receives email and/or SMS invitation with verification link.

2

Order Log Actions

Email Icon: Resend invitation link to borrower
View Timeline: Access detailed order info in Truv Dashboard
Retrieve: Refresh if instant connection made
Import: Add (new record) or Replace (overwrite) URLA employment

3

Document Flow

VOIE report autoflows to Stored Documents. Paystubs and W-2s flow to Draft docs. AUS reference number autoflows to respective AUS order screen.

‚ö†Ô∏è Manual Income Entry Required

Income will NOT be imported automatically. By design, users must review the VOIE report data, analyze, calculate, and manually update the URLA Borrower employment section fields.
